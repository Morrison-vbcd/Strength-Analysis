# 使用手冊 — 訊號判讀與注意事項

> 本手冊不講安裝(見 README.md),只講一件事:**這套工具的訊號什麼時候能信、什麼時候只是暫定,以及如何過濾雜訊**。
> 適用三個分頁:個股相對強弱、板塊輪動(RRG)、配對交易。

---

## 零、三條鐵律

所有誤用幾乎都源自違反這三件事:

1. **顏色與訊號都是「相對」,不是「絕對」。**
   紅格 ≠ 股價下跌,可能只是漲得比族群少;RS 上升 ≠ 股價上漲,可能只是跌得比較少。
   工具回答的是「**選哪一檔**」;「**何時進場**」要回到該股絕對價格自己的結構。

2. **最後一段結構(欄名尾端 `*`)尚未確認,會重繪。**
   ZigZag 的轉折點要等反向波動超過門檻(預設 6%)才成立。進行中的那段,
   方向、長度、標籤、落在上面的 ⭐/⚠️ 都可能隨新資料改變。
   **已收段(沒有 `*` 的欄)不會重繪**,歷史訊號是固定的。

3. **未收盤的週期,一律視為暫定。**
   週線資料要等**當週收盤(週五)**才定;日線要等**當日收盤**才定。
   盤中看到的最新一點永遠是「畫到一半的 K 棒」。

## 一、訊號確認時點對照表

| 訊號 | 暫定期間 | 何時確認 |
|---|---|---|
| 熱力圖最後一欄(`*`) | 該段進行中 | 反向波動 ≥ ZigZag 門檻、出現新轉折後 |
| Pattern ⭐/⚠️ 落在 `*` 欄 | 同上 | 同上(落在已收欄的訊號不會變) |
| RS 創波段新高/新低(最新段) | 同上 | 同上 |
| RRG 週線的最新一點與訊號 | 週五收盤前整週 | 當週收盤後 |
| RRG 日線的最新一點與訊號 | 盤中(報價延遲約 15 分鐘) | 當日收盤後 |
| 配對交易現況 z-score | 盤中 | 當日收盤後 |

工具會直接標示這件事:個股頁的型態訊號清單有「**狀態**」欄(✅已確認 / ⏳進行中),
板塊輪動頁在最新一期未收盤時會顯示 ⏳ 提示。**只依 ✅ 的訊號行動。**

實務建議:**週線 RRG 在週末看、日線訊號在收盤後看**;盤中看到的變化只當「預告」,不當「訊號」。

另外注意**資料快取 15 分鐘**:重跑分析不會立刻抓到最新報價;要強制更新請等 15 分鐘或重啟程式。

## 二、個股相對強弱頁 — 注意事項與訊號過濾

### 2.1 注意事項

- **單格不是訊號。** 一段領漲可能只是一次財報跳空。至少要求「連續兩段同向」或「與 RS 線共鳴」(見 2.3)才視為訊號。
- **上市較晚的標的**早期格子是「資料不足」,跨窗口的累計數字會偏高(公平性限制)。
  要嚴格比較時勾側邊欄「**對齊共同起算日**」。
- **RS 線的位置與交叉取決於回看窗口起點。** 把回看從 24 個月改成 12 個月,「RS 是否 > 1.0」「兩條線在哪交叉」都會變。
  判讀請以「結構」(高低點排列、打底、突破)為主,以「絕對位置」為輔。
- **參數擾動測試**:預設門檻 6% 是經 9 個參考序列實測的穩健值(見 README)。
  一個訊號如果把門檻調到 5% 或 7% 就消失,多半是切段邊界效應,不是真訊號。
- 回看月數太短會讓 leg 樣本不足:**少於 8 段時不要做型態判讀**(24 個月約 10~14 段)。

### 2.2 Pattern 訊號優先級

| 優先級 | 訊號 | 為什麼 |
|---|---|---|
| **最高** | 「**壓力段RS新高**」(內建型態):族群下跌段中 RS 創波段新高 | 在壓力下驗證過的強勢,是熱力圖×RS 線共鳴的內建版 |
| **高** | ▼段「逆勢抗跌」、「抗跌後領漲」、「谷底翻強」+ RS 打底突破同時成立 | 在族群下跌的壓力下驗證過的強勢;或輪動起點的雙重確認 |
| **中** | 「連續領漲」、趨勢中的「RS創波段新高」 | 是**持有確認**,不是進場訊號——趨勢中會重複觸發 |
| **低(單獨不足)** | 單一綠格/紅格、單次「跟漲但落後」 | 單段雜訊,需要後續段驗證 |
| **預警** | 「**領漲背離**」(內建型態):標籤仍領漲、但 RS 段高點一波比一波低 | 力度還在、結構已衰減,通常比「領漲熄火」早一段出現 |

**用「📊 訊號統計」驗證優先級**:熱力圖分頁的訊號統計表會算出**每種型態在你這個族群的歷史應驗率**
(⭐ 訊號應驗 = 下一段贏過族群;⚠️ = 下一段輸給族群;只計已確認樣本,無前視)。
上表是通則,統計表是你這個族群的實證——兩者衝突時,先懷疑樣本數(< 5 不具參考性),再信統計。

空頭側同理:「領漲熄火」單獨只是一段轉弱,**「熄火 + RS 跌破最後一個更高低點」同時出現**才是領導權派發確認。

### 2.3 共鳴過濾(熱力圖 × RS 線)

熱力圖量「每段的力度」,RS 線量「累積的結構」,兩層雜訊來源不同,**同時指向同一方向時假訊號率大幅下降**。一檔股票完整的輪動生命週期,兩層會依序共鳴四次:

```
底部:熱力圖 紅→淡(破底漸變抗跌)  × RS 走平打底            → 停止變弱,列入觀察
啟動:谷底翻強 ⭐                   × RS 打底突破(尤其在▼段)→ 進觀察清單
回測:下一▼段 抗跌                 × RS 回檔守住、更高低點    → 進場級確認
持有:連續領漲                      × RS 新高序列              → 續抱
出場:標籤仍綠但 RS 高點遞減(預警)→ 領漲熄火 × RS 破前低    → 離場
```

**進場前檢查清單**(五題,全部 yes 再動作):
1. 訊號落在**已確認**的 leg 上?(不在 `*` 欄)
2. 有熱力圖 × RS 線**共鳴**?(不是單層單訊號)
3. 門檻調到 5%~7% 訊號**都還在**?
4. 該股**絕對價格**自己的結構允許進場?
5. 用**復盤模式**驗證過:回到類似的歷史轉折點,這類訊號當時出現過、且事後有效?

### 2.4 復盤模式的正確用法與限制

- 復盤會把資料**截斷到基準日**,你看到的就是「當時能看到的」——包括當時尚未確認的 `*` 段。這是驗證訊號最誠實的方式:挑一個歷史轉折,把基準日設在轉折**前**,看訊號是否已出現。
- **步進器**:日期欄下方的「◀ 月 / ◀ 週 / 週 ▶ / 月 ▶」可逐週/逐月推進基準日,
  適合「沿著時間軸走一遍,看訊號怎麼一格一格長出來」。同一季內步進不會重新下載資料,幾乎即時。
- 限制:價格經過除權息調整(auto-adjust),數值與當年即時看到的略有差異;RS 比較不受影響,但絕對價位對不上是正常的。

## 三、板塊輪動(RRG)頁 — 注意事項與訊號過濾

### 3.1 注意事項

- **週線模式下,最新一點在週五收盤前都是暫定**——這是本頁最常見的誤用。週中看到板塊跳象限,先記下,週末收盤後再確認。
- RRG 用滾動窗 z-score 正規化(中心 100),**改「正規化窗/動能回看」參數會改變象限歸屬**。預設值(週線 13/1、日線 63/5)適合大多數情況;調參數時做擾動測試,道理同個股頁。
- 象限只是把連續值切成四格。**貼著中心(RS-Ratio、RS-Mom 都在 100 附近)的板塊,象限轉換多半是雜訊**;離中心越遠的轉換越有意義。
- **「結構強弱熱力圖」分頁的標籤不能照個股頁的讀法讀**:板塊 vs 大盤的單段標籤主要反映
  beta(防禦板塊跌段永遠綠、漲段永遠紅),歷史上綠格下一段勝率僅 29%、紅格 67%——
  意義接近反轉。正確讀法見《策略守則.md》第三之二節。

### 3.2 RRG 訊號過濾

- **看路徑,不只看位置**:用「🎞️ 輪動回放」看板塊是怎麼走到現在的。平滑的順時針弧線 = 正常輪動,可依「落後→改善→領先→轉弱」的節奏操作;來回抖動 = 震盪市,象限訊號失效。
- **罕見轉換權重最高**:訊號表標「罕見」的跳象限(落後→領先直跳、領先→落後急轉)代表劇烈的資金移動,優先關注;常規的相鄰象限轉換則需要下一期確認。
- **逆時針移動是警訊**:改善→落後(而非→領先)代表輪動失敗,該板塊的多頭訊號作廢。
- **用 risk-on/off 做總開關**:防禦板塊(🛡️)集體領先時,進攻板塊的「改善」訊號打折;反之亦然。順著 regime 的訊號才給足權重。
- RRG 的板塊訊號與個股頁共用一個方向時最強:**板塊進「改善/領先」+ 該板塊內個股觸發高優先級 Pattern** = 由上而下共鳴。

## 四、配對交易頁 — 注意事項與訊號過濾

### 4.1 注意事項

- **先看判讀標籤再看 z**:🔴「不建議配對」時,z 再極端都不要做——共整合不成立代表價差沒有回歸的統計理由,可能一路發散。
- 共整合是**歷史性質,會失效**。任一腿發生結構性事件(購併、分拆、商業模式改變)後,過去的檢定作廢,要重新檢驗。建議**每月或持倉前重跑一次**檢定。
- **半衰期是你的預期持倉時間**。半衰期 50 天代表價差收斂一半平均要 50 天——過長(>60 天)資金效率差、途中變數多;過短(<5 天)可能只是雜訊。
- 財報週避開:兩腿各自的財報是價差最大的跳動來源,z 進場訊號落在任一腿財報前一週內,先跳過。

### 4.2 進出場紀律

- 進場:|z| ≥ 進場門檻(預設 2σ)**且收盤確認**,做「多便宜、空昂貴」。
- 出場:z 回到 0 附近平倉;**z 突破停損門檻(預設 3σ)且持續擴大 = 關係可能斷裂,停損離場**,不要凹單等回歸。
- z-score 視窗(預設 60 日)決定「均值」的記憶長度:視窗越短越敏感、越長越遲鈍,調整後記得重新檢視歷史 z 帶是否合理。

### 4.3 歷史回測怎麼讀

頁面下方的「📜 歷史回測」照上述紀律模擬過去的每一筆進出場。讀數字前先知道三個刻意的設計:

1. **β 用滾動 120 日估計**(上方診斷區的 β 是全樣本迴歸,看得到未來;回測不能這樣)。
2. **訊號當日收盤產生、次日收盤成交**——避免「用今天收盤價算訊號又用同一價成交」的偷跑。
   因此回測數字與上方診斷數字**略有出入是正常且正確的**。
3. **不含手續費、滑價、借券成本**。台股融券成本高,實際報酬會明顯低於回測值。

過濾建議:
- **交易筆數 < 5 的回測沒有統計意義**,不要因為一條漂亮的權益曲線就下結論。
- 重點看**停損次數佔比**:停損多 = 這對股票的共整合不穩,z 門檻內的「便宜/昂貴」不可信。
- 權益曲線要的是「樓梯型緩上」;暴衝後大回落 = 報酬集中在一兩筆,運氣成分高。
- 調高進場門檻(2.5σ~3σ)通常筆數變少、單筆變肥;配合半衰期看:門檻越高、等待與持有越久。

## 五、資料限制(所有頁面共用)

- **yfinance 免費源**:報價延遲約 15 分鐘、只有日線;冷門台股偶爾缺資料(工具會提示略過)。
- **Yahoo 限流**:短時間大量請求會被 429 擋下,症狀是「大部分代碼突然抓不到、每次失敗的名單還不一樣」。
  工具已內建逐檔重試與退避,且**嚴重缺漏的結果不會進快取**——看到 📡 限流提示時,等 1~2 分鐘再重跑即可,
  不需要重啟程式。連續狂按重跑只會被擋得更久。
- **除權息調整價**(auto-adjust):RS 比較包含股息再投入效果,高股息標的(如 0056)的 RS 會比純價格比強;跨市場比較時留意。
- **台股 `.TW/.TWO` 自動退補**:輸入純數字代碼即可;若兩者都抓不到會列入略過清單。
- **上櫃/新上市**:歷史短的標的參與比較時,注意公平性(見 2.1)。
- 跨市場代理(如以美元計價 ETF 代理台股主題)會**夾帶匯率漂移**,RS 趨勢仍可參考,幅度不要細讀。

## 六、建議工作節奏

| 頻率 | 做什麼 |
|---|---|
| **每週末**(週線收盤後) | 板塊輪動頁:確認 RRG 象限與訊號表 → 決定下週關注哪些板塊 |
| **每日收盤後** | 個股頁:檢查關注清單的最新欄與 Pattern 訊號;有新訊號 → 過 2.3 的五題檢查清單 |
| **每月一次** | 配對交易:重跑持倉/候選配對的共整合檢定 |
| **進場前** | 復盤模式驗證同類訊號的歷史表現(2.4) |
| **盤中** | 只觀察,不依最新一點行動(鐵律 3) |

## 七、免責聲明

本工具僅呈現歷史價格的相對強弱統計,**不構成投資建議**。所有訊號描述的是「過去在統計上發生了什麼」,不保證未來重現。交易決策與風險自負;進場時機、部位大小、停損請依自己的交易計畫執行。
